User Profile Page

Technical Description

  • HTML Techniques Used:

    • Semantic Tags: Proper use of (<header>, <section>, <div>, <footer>) for better structure.
    • Flexbox Layout: Used flexbox for a responsive and adaptable page layout.
    • Grid Layout for Calendar: Implemented grid-template-columns: repeat(7, 1fr) to structure the calendar.
    • Navigation Bar: <nav> with dropdown menus for structured and accessible navigation.
    • Images and Illustrations: <img> tags for logos, user profile pictures, and event icons with descriptive alt attributes.

  • CSS Techniques Used:

    • Responsive Design: Flexbox and grid layouts adjust the content layout across different screen sizes.
    • Calendar Styling: Grid layout for organizing days of the week , Color-coded dates for marking today's date and scheduled events , Hover effects on calendar days for better interaction.
    • Progress Bars for Skills: .skill_bar and .skill_progress classes define structured skill bars , Smooth animations using transition: width 0.5s; for gradual loading , Tooltips on hover (.skill_tooltip) provide detailed information on skill levels.
    • Dropdown Menus: Styled dropdown navigation with hover effects for better user experience.

  • JavaScript Techniques Used:

    • Dynamic Calendar: Utilized Date() object to generate calendar days dynamically , Event markers using the ::after pseudo-element to highlight scheduled events.
    • Interactive Progress Bars: Dynamically sets width of .skill_progress bars based on predefined skill levels , Tooltip displays on hover for detailed skill descriptions.

Accessibility

  • Semantic HTML: The use of proper <header>, <section>, <nav>, <footer> tags improves screen reader navigation.
  • Color Contrast: The background and text colors are chosen to enhance readability.
  • Alternative Text for Images: Any images used should have alt attributes to describe their content.
  • Hover Effects: Used for interactive elements such as dropdowns, buttons, and tooltips to enhance usability.

Sitemap Page

Technical Description

  • HTML Techniques Used:

    • Semantic Structure: Utilized <header>, <nav>, <svg>, <footer> for proper page organization.
    • Navigation Bar: Implemented <nav> with an unordered list <ul> for structured and accessible navigation.
    • SVG-based Sitemap: Used an <svg> element with <rect> and <line> elements to create a visual sitemap structure.
    • Hyperlinks with Interactive Nodes: <a> elements wrap <rect> nodes to provide clickable navigation to different sections.

  • CSS Techniques Used:

    • Responsive Design: Ensured adaptability across screen sizes, adjusting height for mobile screens using @media queries.
    • Sitemap Styling: .node class for sitemap elements, with smooth hover and transition effects, connection class for structured sitemap links using stroke-width: 2px , Focus effects for improved accessibility using box-shadow for focusable elements , Focus effects for improved accessibility using box-shadow for focusable elements.
    • Navigation Styling: Dropdown navigation with .dropdown and .dropdown-menu , Hover and active effects for a better user experience.
    • Footer Design: Styled with a background color, centered text, and an embedded logo.

  • Key Tags & Properties Justification:

    • .node:hover: Enhanced interactivity and user engagement.
    • .sitemap-svg: Maintained scalability across screen sizes.
    • .nav: Improved navigation usability with consistent padding and color.

Accessibility

  • Keyboard Navigation: Focusable nodes (tabindex="0") allow keyboard users to navigate through the sitemap.
  • Contrast: Dark green background with white text for improved readability.
  • Text for Images:<img> elements include descriptive alt attributes for visually impaired users.

Content Page

Technical Description

  • HTML Techniques Used:

    • Semantic tags: (<header>, <section>, <main>) for better structure.
    • Images and Media: img tags used for visual engagement, with alt attributes for accessibility.
    • Interactive Elements: Links (a) for navigation and button for scroll-to-top functionality.

  • CSS Techniques Used:

    • Color Scheme: Calming and earthy tones, including #C7C9A3 (background) and #6F7149 (accents), consistent with the wellness theme.
    • Hover Effects: Smooth transition effects (transform: translateY(-5px)) on cards for interactive user experience.
    • Responsive Design: Utilized CSS Grid (.quick-tips, .events-grid) for flexible layouts that adapt to different screen sizes.
    • Use background-video for increase interactivity

  • JavaScript Techniques Used:

    • Scroll-to-Top Button: Implemented using window.scrollTo() with smooth scrolling behavior.
    • Dynamic Button Visibility: window.onscroll event listener toggles visibility of the scroll-to-top button.

Accessibility

  • Keyboard Navigation & Focus Management: The "Scroll to Top" button includes aria-label="Scroll to top" for screen readers. The .top-btn:hover, .top-btn:focus styles improve visibility for keyboard users. The tabindex="0" on interactive elements ensures they can be accessed using the Tab key.
  • Color Contrast & Readability: Text colors contrast well with the dark background to enhance readability. Text shadows and backgrounds are used to improve clarity against the background video.
  • Alternative Text for Images: All images include meaningful alt attributes to describe their content.
  • Video Accessibility: The background video is muted by default and does not contain critical content, avoiding distractions for screen readers. A dark overlay improves text readability over the video.
  • ARIA Labels for Interactive Elements: Navigation buttons and external links use aria-labels to describe their purpose.
  • Mobile Responsiveness: The site adjusts dynamically for smaller screens using CSS media queries.

Challenges and Lessons Learned

  • Challenges Encountered:

    • Contrast Issues with Background Video – The video background made text hard to read, requiring a dark overlay and text shadows.
    • Keyboard Navigation Limitations – Some elements, like the scroll-to-top button, were not initially accessible using the Tab key.
    • Screen Reader Accessibility – Some image alt text and interactive elements lacked proper ARIA labels, affecting usability.
    • Mobile Responsiveness – Certain elements did not scale well on smaller screens, requiring grid adjustments.
    • Clickable Elements in SVG Sitemap – SVG nodes were not keyboard-friendly, requiring tabindex="0" and wrapping.
    • Ensuring Proper Focus States – Some interactive elements needed focus indicators to improve visibility for keyboard users.

  • Lessons Learned:

    • Text Readability Must Be Prioritized – Dark overlays, shadows, and high contrast colors improve clarity.
    • Keyboard Navigation is Essential – Adding focusable elements and tabindex="0" enhances usability.
    • Screen Readers Need Clear Labels – Using ARIA attributes ensures better accessibility.
    • Mobile Optimization is Crucial – CSS grid and media queries help maintain a responsive layout.
    • SVGs Require Extra Accessibility Work – Making SVG nodes clickable and focusable improves interaction.
    • Accessibility Testing Prevents Issues – Testing with screen readers and keyboard navigation saves time later.

Compliance

  • Accessibility Compliance

    • The pages follow WCAG (Web Content Accessibility Guidelines) by including proper alt attributes for images, ensuring visually impaired users can access content through screen readers.
    • HTML is structured correctly, using semantic tags such as <header>, <nav>, <section>, and <footer> to improve readability and assistive technology support.
    • Contrast and font sizes are optimized for readability, making the site more inclusive.

  • Security Compliance

    • No inline JavaScript that could lead to security vulnerabilities.
    • External links are reviewed to ensure they point to safe and relevant sources.

  • Content Appropriateness and Ethical Use

    • The site contains nformative content, aligning with Janet’s guidelines on appropriate material.
    • No offensive, misleading, or unauthorized material is published, ensuring compliance with acceptable use policies.

  • Technical Standards Compliance

    • The pages are validated using W3C HTML and CSS validators, ensuring clean and error-free code.
    • The site is optimized for cross-browser compatibility, ensuring proper rendering on different devices and browsers.
