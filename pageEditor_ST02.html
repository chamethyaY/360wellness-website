Home Page

Technical Description

  • HTML Techniques Used:

    • Overall- Semantic tags (<header>, <section>, <div>, <footer>) for better structure.
    • Dropdown Navigation: Implemented a dropdown menu inside <li class="dropdown">.
    • Image & Logo Integration: img elements used for branding and content.
    • Responsive Design: meta name="viewport" content="width=device-width, initial-scale=1.0".

  • CSS Techniques Used:

    • Flexbox Layout: Used display: flex; for navigation and header alignment.
    • Dropdown Menu Styling: Applied position: absolute; and box-shadow for better visibility.
    • Overlay Effect: .overlay { background: rgba(0, 0, 0, 0.5); } for a semi-transparent hero section.
    • Hover Effects: nav ul li a:hover { text-decoration: underline; }.
    • Border Radius for Circular Elements: Applied border-radius: 50%; for volunteer section elements.

  • JavaScript Techniques Used:

    • document.querySelector('.dropdown').addEventListener('click', function() {...})
    • window.addEventListener('scroll', function() {...}).
    • document.getElementById('form').onsubmit = function() {...}

Accessibility

  • Semantic HTML: The use of proper <header>, <section>, <nav>, <footer> tags improves screen reader navigation.
  • Color Contrast: The background and text colors are chosen to enhance readability.
  • Alternative Text for Images: Any images used should have alt attributes to describe their content.

Project Table Page

Technical Description

  • HTML Techniques Used:

    • Table Structure: Used table, thead, body to display structured health initiative data.
    • Navigation Bar: Implemented a nav element with an unordered list ul for site navigation.
    • Dropdown Menu: Used li class="dropdown" to create a multi-level menu with ul class="dropdown-menu".
    • Hyperlinks for Details: Each row has a "Read More" link using a class="read-more" href="#".
    • Header & Footer: Used header for branding and footer for copyright and editor details.

  • CSS Techniques Used:

    • background-color: #007F5B; → Dark green for the table.
    • tr:nth-child(even) { background-color: #6b6e4e; } → Alternating row colors for readability.
    • tr:hover { background-color: #7b7e5e; } → Hover effect on rows.

Accessibility

  • ARIA Labeling & Semantic Structure: The <svg> element includes aria-label="Website Sitemap" to define its purpose for screen readers. <title>, tags inside <rect> elements provide descriptive names for each section.
  • Keyboard Navigation & Focus Managemen: tabindex="0" makes each node focusable using the Tab key. Hover and focus styles ensure visual feedback for keyboard users
  • Screen Reader Compatibility: Each clickable node is wrapped inside <a> elements, ensuring it can be navigated via keyboard and read properly by screen readers.

Challenges and Lessons Learned

  • Challenges Encountered:

    • Making sure the webpage looks good on different screen sizes was challenging. The layout sometimes broke on smaller screens, requiring adjustments with flexbox and media queries.
    • The dropdown menu didn’t function properly at first. It required CSS positioning adjustments and JavaScript enhancements to ensure smooth visibility and interactivity.
    • Creating a visually appealing and readable table was tricky. The background colors, alternating row styles, and hover effects had to be carefully chosen for contrast and accessibility.

  • Lessons Learned:

    • Using section, header, footer, and nav helped create a well-organized and readable webpage, making it easier for search engines and users.
    • Instead of using floats or inline-block, flexbox and grid provided better control over page structure, making it more responsive and visually appealing.
    • The website looked great on a desktop, but on mobile screens, some sections didn’t align properly. Regular testing and debugging helped optimize the user experience.

Compliance

  • Accessibility Compliance

    • The pages follow WCAG (Web Content Accessibility Guidelines) by including proper alt attributes for images, ensuring visually impaired users can access content through screen readers.

    • Contrast and font sizes are optimized for readability, making the site more inclusive.

  • Security Compliance
