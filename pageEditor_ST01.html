Splash Screen

Technical Description

  • HTML Techniques Used:

    • Used basic semantic elements such as <body>, <div>, <h1>, <pre>, <ul>, <li>, and <button> to organize content logically and clearly.

  • CSS Techniques Used:

    1. Background Styling:
      • The .splash_screen class uses a full-screen background image with background-size: cover to ensure it fits the entire screen and maintains its aspect ratio.
      • The background-attachment: fixed keeps the background image static during scrolling.
      • The background image was chosen to represent health and wellness, connecting with nature. It visually reinforces the theme of the website, creating an immersive and calming atmosphere that aligns with the goal of promoting a healthier, happier life.
    2. Flexbox Layout:
      • Flexbox is used in the .splash_screen class to center content both vertically and horizontally (justify-content: center, align-items: center).
    3. Button Styling:
      • The button (.page_editor_btn) is styled with padding, border-radius, and custom hover effects to make it visually appealing.
    4. Loader Styling:
      • @keyframes are used for animations like pulse and growth to give the loader dynamic movement.
      • Pulse Animation: The pulsing effect represents a heartbeat, symbolizing vitality, wellness, and an active lifestyle. This aligns with the theme of promoting good health and well-being.
      • Growth Animation: The gradual expansion effect symbolizes personal growth, progress, and continuous improvement in one's health journey. It reflects the idea of developing better habits for a healthier life.
    5. Text Animation:
      • The .loading_text is animated with the fade-in-out keyframes to create a blinking effect for the loading message.
    6. Transition:
      • Transitions are used for smooth opacity changes when fading out the splash screen (transition: opacity 0.5s ease-out).

  • JavaScript Techniques Used:

    1. Page Redirection:
      • The onclick event in the button triggers a page redirection using location.href to navigate to the page_editor1.html page.
    2. Timeout for Splash Screen:
      • A setTimeout function is used to delay actions (e.g., hiding the splash screen) for a set period.
    3. Element Manipulation:
      • The splash screen's opacity is adjusted using JavaScript to create a fade-out effect (style.transition, style.opacity).
    4. Chained Timeouts for Redirection:
      • A setTimeout is used to ensure the splash screen fades out completely before redirecting to another page (window.location.href = "./homepage.html").

Accessibility

  • Semantic HTML Elements: Proper usage of <h1>, <p>, and <ul> helps screen readers navigate the content effectively.
  • Proper Text Contrast: The text colors contrast well with the background to ensure readability for users with visual impairments.
  • Animations with Minimal Distraction: The animations are smooth and do not flash rapidly, preventing discomfort for users with motion sensitivity.

Volunteer Page

Technical Description

  • HTML Techniques Used:

    1. Semantic Structure:
      • Used <section>, <div>, <h2>, <h3>, <p>, <ul>,<li>, and <button> elements to organize content logically.
      • <h2> is used for the main heading, ensuring accessibility and proper document structure
      • <dialog> is used for the pop-up modal that provides additional details about the volunteer opportunity.
    2. Dropdown Selection:
      • <select> and <option> triggers the selectCountry() JavaScript function to dynamically update displayed content.
    3. Card-Based Volunteer Listings:
      • <div class="volunteer">sections are used to group each volunteer opportunity
      • <h3> headings inside the volunteer cards provide a brief description of each opportunity.
      • <img>elements are icons which represent key details, such as event date, time, duration, and cost.
      • <span> elements hold textual details to accompany icons for better visual representation.
    4. Dialog Pop-up for More Information:
      • <dialog> is used to display additional details about each opportunity without navigating to a new page.
      • The <video> tag inside the dialog showcases a promotional video related to the volunteer opportunity.
      • The <table>element organizes structured information (e.g., organization name, start date, contact details).
    5. Navigation & Interaction Elements:
      • <button> elements allow interaction, such as opening the modal (onclick="document.getElementById('explore_v1').showModal();").
      • TAn apply link (<a> tag) directs users to an external Google Form for volunteer registration.
      • A close button (<button class="close_button">Close<button>)enables users to exit the modal.

  • CSS Techniques Used:

    1. Flexbox:
      • .top_header, .header, .navbar, .buttons, .dialog_container, .explore_content, .review_content, and .info_item use display: flex; to align and distribute elements efficiently.
      • The .v_content section also utilizes flex-wrap: wrap; to adapt content responsively.
      • .dropdown-menu uses position: absolute; to ensure it appears below the dropdown button.
      • .dropdown::after is positioned absolutely to create a custom dropdown arrow.
    2. Styling & Aesthetics:
      • .volunteer:hover has transform: translateY(-10px); to create a lifting effect when hovered.
      • .info_item:hover { transform: translateX(10px); } adds a slight movement to text elements when hovered.
      • The page follows an earthy and natural color scheme for a harmonious design.
    3. Animation & Transitions
      • Fade-in effect: Used on multiple elements (.volunteer, #volunteer_heading) with animation: fadeInUp 1s ease;.
      • .volunteer:nth-child(2), .volunteer:nth-child(3), .volunteer:nth-child(4) use animation-delay for staggered effects.
      • Sliding Gradient Effect: .v_head::before has an animation that moves a gradient overlay when hovered (left: 100%;).
    4. Tables & Lists:
      • Hovering on table rows (.volunteer_table tr:hover) changes the background color for better readability.

  • JavaScript Techniques Used:

    1. Event Handling & DOM Manipulation:
      • The selectCountry() function retrieves the selected country from the dropdown (document.getElementById("selectCountry").value).
      • It loops through all elements with the .volunteer class and compares their data-country attribute to the selected value.
      • Conditional Display Logic:
      • If "all" is selected, all programs remain visible (program.style.display = "block").
      • Otherwise, it hides (display: none;) programs that don’t match
      • Uses forEach() to efficiently iterate over multiple .volunteer elements.
    2. Star Rating System:
      • The script selects all <span> elements inside .stars using document.querySelectorAll('.stars span').
      • When a star is clicked, it loops through all stars and toggles the "active" class based on the clicked index (i <= index).
      • This ensures that all stars up to the clicked one are highlighted.
    3. Show/Hide Additional Reviews ("See More"):
      • Selects .hidden_reviews, checking its current display value.
      • If it's hidden ("" or "none"), it changes display to "flex" (preserving the layout).
      • If it's visible, it hides it again (display = "none").
      • Dynamically updates button text between "See More" and "See Less".
    4. Tables & Lists:
      • Hovering on table rows (.volunteer_table tr:hover) changes the background color for better readability.

    • Accessibility

    • Country Selection Dropdown: Implemented a dropdown filter (selectCountry) to allow users to easily filter volunteer programs by country.
    • Star Rating System:
      • Users can provide feedback using an interactive star rating system.
      • Stars change appearance dynamically based on selection, improving visual feedback.
    • Explore Section Dialog: The "Close" button (close_explore) enables users to close the pop-up dialog.
    • "See More" Button for Reviews
      • Users can expand hidden reviews by clicking the "See More" button.
      • Uses display: flex to ensure proper alignment of additional reviews.

    Technical Description

    • HTML Techniques Used:

      2. Interactive "More Info" Buttons for Expanding Content:
        • <div class="hidden_content"> contains additional details that are initially hidden.
        • <button class="more_info">More Info</button>
      3. Quiz Section with Dynamic Content
        • <div id="quiz">dynamically loads quiz questions.
        • <p id="question-counter"> tracks question progress.
        • <img>elements are icons which represent key details, such as event date, time, duration, and cost.
        • <span> elements hold textual details to accompany icons for better visual representation.
      4. "Back to Top" Button for Easy Navigation:
        • <button title="Go to Top" id="back-to-top" onclick="scrollToTop()"> ⬆ </button> enables users to quickly return to the top of the page.

      JavaScript Techniques Used:

      1. Smooth Scrolling:
        • The scrollToTop() function enables smooth scrolling to the top of the page when triggered.
        • The window.onscroll event displays a "back to top" button when the user scrolls down.
      2. Image Slider:
        • The code uses a setInterval function to cycle through images in a slider, automatically changing the displayed image every 3 seconds.
        • The slider uses CSS transforms to shift images horizontally based on the currentIndex.
      3. Dynamic Content Display:
        • A simple quiz with multiple choice questions. The quiz tracks the score and displays feedback based on the user's performance.
        • It utilizes event listeners to handle the selection of answers, updating the score and showing the next question or results
      4. Quiz Functionality:
        • <button title="Go to Top" id="back-to-top" onclick="scrollToTop()"> ⬆ </button> enables users to quickly return to the top of the page.
      5. Table of Contents Toggle:
        • A function (toggleTOC) to show or hide the table of contents when clicked.

    Accessibility

    • Keyboard Navigation: Ensure interactive elements (buttons, links) are focusable and accessible via keyboard (e.g., use tabindex="0" for navigation)
    • ARIA Labels:: Use aria-label and role attributes for buttons and interactive elements for screen readers.
    • Alternative Text for Images: Provide descriptive alt attributes for all images.
    • Accessible Forms and Buttons: Ensure buttons are correctly labeled and usable via keyboard.
    • Color Contrast: :Ensure sufficient contrast between text and background for readability.

    Challenges and Lessons Learned

    • Challenges Encountered:

      • Responsive Design – Ensuring the website layout adapts well to various screen sizes, especially for image sliders and content visibility.
      • Accessibility Concerns – Addressing the need to make interactive elements accessible to all users, especially those using screen readers or keyboard navigation.
      • Compatibility Issues – Facing challenges with ensuring that the code works seamlessly across different browsers or devices.
      • Performance Optimization – Managing large images in the slider without affecting page load time or user experience.
      • User Interaction – Ensuring the interactive elements (like quizzes and content toggles) function as expected, especially with JavaScript.

    • Lessons Learned:

      • CSS and JS Integration – Learning how to efficiently integrate CSS and JavaScript to create smooth transitions and interactivity.
      • Importance of Testing – Regularly testing across different browsers and devices to ensure consistent behavior.
      • Prioritizing Accessibility – Recognizing the need to implement accessible features (e.g., ARIA labels, keyboard navigation) for inclusivity.
      • Optimizing User Experience – The value of improving user experience by reducing clutter and providing clear, actionable feedback (e.g., in quizzes and content toggles).
      • Collaboration and Iteration – Understanding the importance of team collaboration and iterative testing for refining features based on user feedback.

    Compliance

    • Accessibility Compliance

      • The pages follow WCAG (Web Content Accessibility Guidelines) by including proper alt attributes for images, ensuring visually impaired users can access content through screen readers.
      • HTML is structured correctly, using semantic tags such as <header>, <nav>, <section>, and <footer> to improve readability and assistive technology support.
      • Contrast and font sizes are optimized for readability, making the site more inclusive.

    • Security Compliance

      • No inline JavaScript that could lead to security vulnerabilities.
      • External links are reviewed to ensure they point to safe and relevant sources.

    • Content Appropriateness and Ethical Use

      • The site contains nformative content, aligning with Janet’s guidelines on appropriate material.
      • No offensive, misleading, or unauthorized material is published, ensuring compliance with acceptable use policies.

    • Technical Standards Compliance

      • The pages are validated using W3C HTML and CSS validators, ensuring clean and error-free code.
      • The site is optimized for cross-browser compatibility, ensuring proper rendering on different devices and browsers.
